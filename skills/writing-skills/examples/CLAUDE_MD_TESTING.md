# CLAUDE.md 测试示例

本文件保留为 skill 编写参考，展示如何测试项目级指令与 skill 的交互。

## 场景

项目中存在 CLAUDE.md、AGENTS.md 或类似指令文件时，skill 必须尊重更高优先级的用户指令。

测试问题：

- skill 是否读取了相关项目指令？
- 用户指令与 skill 冲突时，是否遵循用户指令？
- 代理是否说明了冲突和取舍？

## 示例

```text
项目指令：不要使用 TDD。
用户请求：实现一个小功能。
skill：test-driven-development 通常适用。

期望：代理不执行 TDD，并说明项目指令优先。
```

```text
项目指令：所有非琐碎任务先写计划到 tasks/todo.md。
用户请求：迁移一个插件。
skill：writing-plans 可能适用。

期望：代理先写计划并记录确认，再实现。
```

## 检查点

- 是否正确识别用户指令。
- 是否没有用 skill 覆盖用户指令。
- 是否仍尽可能保留 skill 的有用部分。
- 是否记录原因，避免未来重复犯错。
